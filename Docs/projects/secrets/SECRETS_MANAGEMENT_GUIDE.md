# Secrets Management Strategy for Agentis

## Current Problems
- API keys mixed with config in `.env` files
- No encryption at rest
- Difficult to rotate secrets
- No audit trail
- Hard to manage across environments

## Recommended Solution: Phased Approach

### Phase 1: SOPS + Age (Immediate)

Mozilla SOPS (Secrets OPerationS) with Age encryption provides:
- File-based encryption that works with Git
- Selective encryption (only secrets, not config)
- Simple key management
- CI/CD integration

#### Setup SOPS

1. **Install SOPS and Age**
```bash
# macOS
brew install sops age

# Linux
curl -LO https://github.com/mozilla/sops/releases/download/v3.8.1/sops-v3.8.1.linux.amd64
chmod +x sops-v3.8.1.linux.amd64
sudo mv sops-v3.8.1.linux.amd64 /usr/local/bin/sops
```

2. **Generate Age Key**
```bash
# Generate a new key pair
age-keygen -o ~/.config/age/keys.txt

# The public key will be displayed, save it
```

3. **Create SOPS Config**
```yaml
# .sops.yaml
creation_rules:
  - path_regex: secrets.*\.ya?ml$
    age: age1... # your public key here
  - path_regex: \.env\.secrets$
    age: age1... # your public key here
```

#### Restructure Secrets

**Before (current):**
```
.env.prod  # Contains everything mixed together
```

**After (recommended):**
```
.env.config          # Non-sensitive configuration
.env.secrets.enc     # Encrypted secrets only
.env.local          # Local overrides (gitignored)
```

**3. Encrypt the secrets:**
```bash
sops -e .env.secrets > .env.secrets.enc
rm .env.secrets  # Remove unencrypted version
```

**4. Create loading script:**
```bash
#!/bin/bash
# scripts/load-secrets.sh

# Decrypt secrets to temporary file
sops -d .env.secrets.enc > .env.secrets.tmp

# Combine config and secrets
cat .env.config .env.secrets.tmp > .env

# Clean up
rm .env.secrets.tmp

echo "Environment loaded with decrypted secrets"
```

#### Integration with Docker

**Update docker-compose files:**
```yaml
# docker-compose.prod.yml
services:
  api:
    env_file:
      - .env  # This will be generated by load-secrets.sh
    # ... rest of config
```

**Update deployment script:**
```bash
#!/bin/bash
# Deploy script
./scripts/load-secrets.sh
docker-compose -f docker-compose.prod.yml up -d
```

### Phase 2: Vault for Production (Future)

When you scale up, migrate to HashiCorp Vault:

```yaml
# docker-compose.vault.yml
version: '3.8'
services:
  vault:
    image: vault:latest
    cap_add:
      - IPC_LOCK
    environment:
      VAULT_DEV_ROOT_TOKEN_ID: myroot
      VAULT_DEV_LISTEN_ADDRESS: 0.0.0.0:8200
    ports:
      - "8200:8200"
    volumes:
      - vault_data:/vault/data
      - vault_config:/vault/config
```

**Benefits of Vault:**
- Dynamic secret generation
- Automatic rotation
- Audit logging
- Fine-grained permissions
- API integration

## Best Practices

### 1. Secret Classification
```
PUBLIC     -> .env.config (committed)
SENSITIVE  -> .env.secrets.enc (committed, encrypted)
LOCAL      -> .env.local (gitignored)
```

### 2. Key Management
- Use separate keys per environment
- Store Age keys securely (password manager)
- Rotate keys quarterly
- Document key recovery process

### 3. Rotation Strategy
```bash
# Monthly rotation script
#!/bin/bash
echo "Rotating secrets..."

# Generate new keys
NEW_JWT_SECRET=$(openssl rand -hex 32)
NEW_CREDS_KEY=$(openssl rand -hex 32)

# Update secrets file
sops --set '["JWT_SECRET"]' "$NEW_JWT_SECRET" .env.secrets.enc
sops --set '["CREDS_KEY"]' "$NEW_CREDS_KEY" .env.secrets.enc

echo "Secrets rotated. Restart services."
```

### 4. Access Control
- Limit who has Age private keys
- Use separate keys for dev/staging/prod
- Audit secret access regularly

### 5. Monitoring
- Log secret decryption events
- Monitor for unauthorized access
- Alert on failed decryptions
