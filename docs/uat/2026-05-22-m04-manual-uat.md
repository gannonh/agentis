# M04 Manual UAT — Projects, Context, and Documents

**UAT scope:** M04 acceptance criteria on branch `feat/M04` (local dev with `pnpm dev`).

**Spec:** Historical M04 projects, context, and documents spec in `docs/specs/_done/`.

**Sign-off:** Pending — reply `accept` or `reject` (note any failed criterion).

---

## Prerequisites

1. Checkout branch `feat/M04`.
2. Ensure repo root `.env` includes M04 storage settings (see `.env.example`):
   - `AGENTIS_STORAGE_ROOT`
   - `AGENTIS_DOCUMENT_MAX_UPLOAD_BYTES`
   - `AGENTIS_DOCUMENT_PREVIEW_MAX_CHARS`
   - `AGENTIS_PROJECT_GOALS_MAX_CHARS`
   - `AGENTIS_PROJECT_MEMORY_MAX_CHARS`
3. Start the stack with mock runtime (predictable document generation):

   ```bash
   AGENTIS_MOCK_RUNTIME=1 pnpm dev
   ```

4. Open **http://localhost:5173** in a browser.

---

## 1. User can create a project and start a thread inside it

1. Go to **http://localhost:5173/projects/new** (sidebar: **Projects** → **+**, or create flow).
2. Enter **Project name** (required), optional **Description** and **Goals**.
3. Click **Create project**.

**Expected:**

- Redirect to `/threads/new?projectId=…`.
- **Project** dropdown in the composer shows your new project (not “No project”).
- Left sidebar **Projects** lists the new project and it is **selected** (highlighted).

4. In the composer, enter a prompt (e.g. `Summarize next steps`) and send.

**Expected:**

- URL becomes `/threads/…`.
- Thread loads without a project error.
- Sidebar keeps the same project selected while the thread is open.

---

## 2. User can edit project details and manage enabled project memories

1. From the thread page, open the project link in the **Project context** card, or go to `/projects/<projectId>` (id from the new-thread URL query or sidebar project link).
2. Change **Name**, **Description**, or **Goals** → **Save changes**.

**Expected:** Values persist after reload.

3. Under **Project memories**, add a memory → confirm it appears.
4. Uncheck **Enabled for context** on one memory, edit another’s text (blur to save), delete one.

**Expected:** List updates; enabled/disabled state sticks after reload.

---

## 3. Project goals and enabled memories are included in run context and visible from the thread page

1. On the project page, set **Goals** (e.g. `Ship the beta by June`) and ensure at least one memory is **enabled** (e.g. `Use a professional tone`).
2. Start a new thread in that project (project selected in dropdown) or use an existing project thread.
3. Send any prompt and wait for the run to finish.

**Expected on the thread page:**

- Right sidebar: **Project context** with project name, **Goals**, and **Enabled memories**.
- Run timeline includes a step like **Project context loaded** (with project/memory metadata).

**Optional API check:**

```bash
curl -s http://127.0.0.1:3001/api/threads/<threadId> | jq '.projectContext'
```

**Expected:** `project`, `goals`, `memories`, and `enabledMemoryCount` populated; disabled memories omitted.

---

## 4. A run can create at least one durable document linked to the run, thread, project, and Library

1. In a **project-scoped** thread, send:

   ```text
   Write a project report document for this project
   ```

   Words like `brief`, `document`, `notes`, `playbook`, or `report` trigger mock document creation when `AGENTIS_MOCK_RUNTIME=1`.

**Expected on the thread:**

- Assistant message mentions a created document (e.g. **Project report**).
- Timeline: **Document created: …**

2. Open **Library** — **http://localhost:5173/library**.

**Expected:** Card for that document with project/thread provenance.

**Optional API check:**

```bash
curl -s http://127.0.0.1:3001/api/documents | jq '.[0] | {title, projectId, threadId, runId}'
```

**Expected:** Non-null `projectId`, `threadId`, and `runId` for the generated document.

---

## 5. User can upload a document to the Library

1. On **Library**, click **Upload document**.
2. Set **Title**, **Type** (e.g. Document), optional **Project**, choose a small file (e.g. `.md` or `.txt`).
3. Click **Upload**.

**Expected:** Dialog closes; new card appears in the grid (may take a moment for the list to refresh).

---

## 6. User can browse, search, filter, and download documents from the Library

1. On **Library**, confirm multiple cards (generated + uploaded) if you completed criteria 4 and 5.

2. **Search:** Type part of a document title in **Search documents**.

   **Expected:** List narrows; **Reset filters** clears it.

3. **Filter by type:** Choose e.g. `markdown`.

   **Expected:** Only matching types shown.

4. **Filter by project:** Choose your project.

   **Expected:** Only documents for that project.

5. **Download:** Click **Download** on a card.

   **Expected:** Browser downloads a file; no error under the button.

---

## 7. Archiving a project hides it from new-thread selection while preserving document ownership and provenance

1. Open `/projects/<projectId>` for a project that has documents (from criteria 4–5).
2. Click **Archive project**.

**Expected:**

- Redirect to new thread.
- Project page shows **Archived** if opened again.
- Archived project **no longer appears** in the left sidebar project list (active projects only).

3. Go to **http://localhost:5173/threads/new**.

**Expected:** **Project** dropdown does not list the archived project (only “No project” and other **active** projects).

4. Return to **Library**.

**Expected:** Documents still visible; provenance still shows the archived project name (e.g. `Your Project · Thread title`).

5. **Optional API check** — creating a thread on an archived project should fail:

```bash
curl -s -X POST http://127.0.0.1:3001/api/threads \
  -H 'Content-Type: application/json' \
  -d '{"prompt":"test","projectId":"<archivedProjectId>"}' | jq .
```

**Expected:** `400` with `code: "project_archived"`.

---

## 8. Thread, project, document, and storage state survive app restart

1. Note one **thread URL**, **project id**, and a **document title** from Library.
2. Stop dev (`Ctrl+C` in the terminal running `pnpm dev`).
3. Start again:

   ```bash
   AGENTIS_MOCK_RUNTIME=1 pnpm dev
   ```

4. Reload the thread URL, project page, and Library.

**Expected:**

- Thread transcript and project context still present.
- Project (including archived status) and memories unchanged.
- Library documents still listed; **Download** still works.

5. **Optional** — confirm files on disk:

```bash
ls -la ./data/storage/documents/
ls -la ./apps/api/data/agentis.db   # or path from DATABASE_URL in .env
```

**Expected:** Storage directory contains document blobs; SQLite DB still has your rows.
